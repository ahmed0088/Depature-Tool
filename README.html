Setup
01
Firebase — Create Project
Go to console.firebase.google.com → Add project → Name it → Enable Realtime Database (Europe-west1, test mode). Credentials already configured in firebase-config.js for Ibis Dubai.
02
Set the Hotel ID
Open firebase-config.js and set HOTEL_ID to any unique string. All colleagues sharing the same ID share one live dataset. For a second property, just use a different ID.
03
Host the Files
Drop all files into GitHub Pages, Netlify, Vercel, or any static host. No build step. Share the URL with your team — everyone uses the same link, no login required.
04
Firebase Security Rules
In Firebase Console → Realtime Database → Rules, set read/write to true under hotels/$hotel_id. This keeps each property's data isolated if you ever add more properties.

Guest Photos on Departure Cards
1
Take a screenshot of the guest photo
Open Outlook or any window with the guest photo. Use Win+Shift+S or Print Screen to capture.
2
Click the photo zone on the card
The app will display "Ready — press Ctrl+V to paste". Click the zone first, then paste.
3
Press Ctrl+V
Image is compressed and saved to Firebase automatically. You can also drag and drop an image file directly onto the photo zone. Click to zoom, hover for the ✕ remove button.
Keyboard Shortcuts
Escape Close any open modal, drawer, or photo zoom
Ctrl+V Paste screenshot onto a departure card photo zone (click zone first)
Ctrl+A → C Select all text in Opera report, copy, then paste into the app
Backup & Restore
Use the 💾 Export button in the top bar to download a full JSON snapshot of all live data. Use 📂 Import to restore from any backup file — the page reloads after import and all data is live immediately.
